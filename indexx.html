Что такое php?

PHP (рекурсивный акроним PHP: Hypertext Preprocessor) - это очень популярный язык с открытым исходным кодом, особенно подходящий для веб-разработки, который может быть встроен в HTML.
Хорошо, но что это значит на самом деле? Пример сделает все более понятным:

<!DOCTYPE html> <html> <head> <title>пример</title> </head> <body> <?php echo "Привет, я PHP скрипт!"; ?> </body> </html>

Самое лучшее в использовании PHP - это его чрезвычайная простота для новичков, но при этом он предлагает множество расширенных возможностей для профессиональных программистов. Не бойтесь читать длинный список возможностей PHP. Уже через несколько часов вы сможете начать писать свои первые сценарии.

Что я могу сделать с помощью PHP?

Что угодно. PHP в первую очередь ориентирован на создание сценариев на стороне сервера, поэтому вы можете делать все то, что может делать другая программа CGI, например, собирать данные из форм, генерировать страницы с динамическим содержимым или отправлять и получать печенье. Но PHP может делать гораздо больше.
Существует три основных области, в которых используются PHP-скрипты.

  • Сценарии на стороне сервера. Это наиболее традиционная область и основное направление. Для этого необходимы три вещи: парсер PHP (CGI-модуль или сервер), веб-сервер и веб-браузер. Вам необходимо запустить сервер с подключенной установкой PHP. Вы можете получить доступ к выводу PHP-программы с помощью браузера, просматривая PHP-страницу через сервер. Все это может быть запущено на вашей машине, если у вас есть опыт программирования на PHP. См. раздел инструкции по установке для получения дополнительной информации.
  • Создание сценариев из командной строки. Вы можете создать PHP-скрипт и запустить его без необходимости использования сервера или браузера. Чтобы использовать его таким образом, вам нужен только парсер PHP. Этот тип использования идеально подходит для сценариев, которые регулярно запускаются с помощью cron (на *nix или Linux) или планировщика задач (на Windows). Эти скрипты также можно использовать для выполнения простых задач по обработке текстов. См. раздел ание PHP в командной строке для получения дополнительной информации.
  • Написание настольных приложений. PHP, вероятно, не самый подходящий язык для создания настольных приложений с графическим интерфейсом, но если вы хорошо знаете PHP и хотите использовать некоторые расширенные возможности PHP в приложениях на стороне клиента, вы можете использовать PHP-GTK для написания таких программ. Таким образом можно писать платформонезависимые приложения. PHP-GTK - это расширение для PHP, недоступное в основном дистрибутиве.
Теги PHP

Когда PHP анализирует файл, он ищет открывающий и закрывающий теги, которые имеют вид , и которые указывают PHP, где начинать и заканчивать интерпретацию кода. Этот механизм позволяет встраивать PHP в любые документы, поскольку все, что выходит за рамки открывающих и закрывающих тегов PHP, будет проигнорировано синтаксическим анализатором.

Если файл содержит только код PHP, предпочтительнее опустить закрывающий тег PHP в конце файла. Это предотвращает добавление пробельных символов или случайных новых строк после закрывающего тега, что приведет к нежелательным последствиям, поскольку PHP начнет буферизацию вывода, когда программист не намеревался отправлять какой-либо вывод на данном этапе сценария.

<?php echo "Здравствуй мир"; // ... дополнительный код echo "Последнее заявление"; // скрипт заканчивается здесь без закрывающего тега PHP
Комментарии в PHP

Комментарии в стиле "одной строки" комментируют только до конца строки или текущего блока PHP-кода, в зависимости от того, что наступит раньше. Это означает, что HTML-код после // ... ?> o # ... ?> БУДЕТ напечатан: ?> выходит из режима PHP и возвращается в режим HTML, поэтому // или # не могут повлиять на это. Если включена директива конфигурации asp_tags, она действует так же, как // %> и # %>. Однако тег не выходит из режима PHP при однострочном комментарии.

<?php echo 'Это тест'; // Это однострочный комментарий в стиле c++. /* Это многострочный комментарий и еще одна строка комментариев */ echo 'Это еще один тест'; echo 'Последнее испытание'; # Это однострочный комментарий в стиле консоли ?>
Переменные

В PHP переменные представлены знаком доллара, за которым следует имя переменной. Имя переменной чувствительно к регистру. Имена переменных подчиняются тем же правилам, что и другие теги в PHP. Правильное имя переменной должно начинаться с буквы или символа подчеркивания, за которым следует любое количество букв, цифр и знаков подчеркивания.

По умолчанию переменные всегда присваиваются по значению. Это означает, что когда выражение присваивается переменной, все значение исходного выражения копируется в целевую переменную. Это означает, что, например, после присвоения значения одной переменной другой, изменения, внесенные в одну из этих переменных, не повлияют на другую.

Типы переменных в PHP

PHP поддерживает десять примитивных типов.
ре скалярных типа:

  • булево
  • целое число
  • плавающий (число с плавающей точкой, также известное как двойной)
  • строка

Четыре типа композитов:

  • массив
  • объект
  • вызываемый объект
  • итерабельность
Булевы

Это самый простой тип. Булево выражает значение, указывающее на истинность. Он может быть истинным или ложным.

Синтаксис

Константы true или false используются для указания булевого литерала. Оба варианта не чувствительны к регистру.

<?php // == - это оператор, который проверяет на // равенство и возвращает булево значение if ($действие == "показать_версию") { echo "Версия - 1.23"; } // это не обязательно... if ($показать_версию == TRUE) { echo "<hr> } // ...потому что вы можете просто написать его: if ($показать_версию) { echo "<hr>"; } ?>
Целые числа

Целое число - это число из множества ℤ = {..., -2, -1, 0, 1, 2, ...}.

Синтаксис

Целые числа могут быть заданы с использованием десятичной (основание 10), шестнадцатеричной (основание 16), восьмеричной (основание 8) или двоичной (основание 2) нотации, перед которой по желанию ставится знак (- или +). Двоичные целочисленные литералы доступны начиная с версии PHP 5.4.0. Чтобы использовать восьмеричную систему счисления, перед числом ставится 0 (ноль). Чтобы использовать шестнадцатеричную систему счисления, перед числом ставится 0x. Чтобы использовать двоичную систему счисления, в качестве префикса используется число 0b.

<?php $a = 1234; // десятичное число $a = -123; // отрицательное число $a = 0123; // восьмеричное число (эквивалентно 83 десятичному) $a = 0x1A; // шестнадцатеричное число (равно 26 десятичным) $a = 0b111111111111; // двоичное число (равно 255 десятичным) ?>
Плавающие числа

Числа с плавающей точкой или числа с плавающей запятой (также известные как "float", "double" или "real") могут быть определены следующими синтаксисами:

Числа с плавающей точкой имеют ограниченную точность. Хотя это зависит от операционной системы, в PHP обычно используется формат двойной точности IEEE 754, дающий максимальную относительную ошибку округления порядка 1.11e-16. Неэлементарные арифметические операции могут давать большие ошибки, и, разумеется, необходимо принимать во внимание распространение ошибок при совместном использовании нескольких операций

Синтаксис

<?php $a = 1.234; $b = 1.2e3; $c = 7E-10; $d = 1_234.567; // начиная с PHP 7.4.0 ?>
Струны

Строка (тип string) - это набор символов, где символ - это то же самое, что и байт. Это значит, что PHP поддерживает ровно 256 различных символов, а также то, что в PHP нет встроенной поддержки Unicode.

В 32-битных системах и в более ранних версиях PHP, строки (string) не могут быть более 2 ГБ (2147483647 байт).

Синтаксис

<?php echo 'это простая строка'; echo 'Также вы можете вставлять в строки символ новой строки вот так, это нормально'; // Выводит: Однажды Арнольд сказал: "I'll be back" echo 'Однажды Арнольд сказал: "I\'ll be back"'; // Выводит: Вы удалили C:\*.*? echo 'Вы удалили C:\\*.*?'; // Выводит: Вы удалили C:\*.*? echo 'Вы удалили C:\*.*?'; // Выводит: Это не будет развёрнуто: \n новая строка echo 'Это не будет развёрнуто: \n новая строка'; // Выводит: Переменные $expand также $either не разворачиваются echo 'Переменные $expand также $either не разворачиваются'; ?>
Массив

На самом деле массив в PHP - это упорядоченное отображение, которое устанавливает соответствие между значением и ключом. Этот тип оптимизирован в нескольких направлениях, поэтому вы можете использовать его как собственно массив, список (вектор), хеш-таблицу (являющуюся реализацией карты), словарь, коллекцию, стек, очередь и, возможно, что-то ещё. Так как значением массива может быть другой массив PHP, можно также создавать деревья и многомерные массивы.

Синтаксис

Массив (тип array) может быть создан языковой конструкцией array(). В качестве параметров она принимает любое количество разделённых запятыми пар key => value (ключ => значение).

array( ключ => значение, ключ2 => значение2, ключ3 => значение3, ... )

Запятая после последнего элемента массива необязательна и может быть опущена. Обычно это делается для однострочных массивов, то есть array(1, 2) предпочтительней array(1, 2, ). Для многострочных массивов с другой стороны обычно используется завершающая запятая, так как позволяет легче добавлять новые элементы в конец массива.

Объекты

Для создания нового объекта, используйте выражение new, создающее в переменной экземпляр класса:

Синтаксис

<?php class foo { function do_foo() { echo "Код foo."; } } $bar = new foo; $bar->do_foo(); ?>
константы

Константа - это идентификатор (имя) для простого значения. Как следует из названия, их значение не может изменится в ходе выполнения скрипта (кроме магических констант, которые на самом деле не являются константами). Константы чувствительны к регистру. По принятому соглашению, имена констант всегда пишутся в верхнем регистре.

Имя константы должно соответствовать тем же правилам именования, что и другие имена в PHP. Правильное имя начинается с буквы или символа подчёркивания, за которым следует любое количество букв, цифр и символов подчёркивания.

<?php // Правильные имена констант define("FOO", "что-то"); define("FOO2", "что-то ещё"); define("FOO_BAR", "что-то большее"); // Неправильные имена констант define("2FOO", "что-то"); // Это верное объявление, но лучше его не использовать: // PHP однажды может зарегистрировать волшебную константу, // которая нарушит работу скрипта define("__FOO__", "что-то"); ?>
